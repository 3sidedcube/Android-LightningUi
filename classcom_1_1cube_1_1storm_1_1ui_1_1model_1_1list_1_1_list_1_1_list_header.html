Detailed Description

This is the model used when displaying the list in the adapter. The list model is processed into a list of subviews from its children and a com.cube.storm.ui.model.list.List.ListHeader and com.cube.storm.ui.model.list.List.ListFooter from its header and footer. The reason why it's done like this is because we don't want to track a single object in a list which is referenced in multiple places, so we instantiate 2 'place holder' models to handle it. com.cube.storm.ui.model.list.List isn't actually a visible view, where as it's members is.
